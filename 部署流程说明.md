每次修改文件后，在项目根目录执行以下命令：

### 方法一：完整流程


## 🛠️ 常用Git命令

### 查看状态

```bash
# 查看当前修改状态
git status

# 查看提交历史
git log --oneline -10
```

### 撤销操作

```bash
# 撤销工作区的修改（未add）
git checkout -- 文件名

# 撤销已add的文件
git reset HEAD 文件名

# 撤销最后一次提交（保留修改）
git reset --soft HEAD^
```

### 分支操作

```bash
# 查看所有分支
git branch -a

# 创建新分支
git checkout -b 新分支名

# 切换分支
git checkout 分支名
```

## 📞 问题排查

### 推送失败

```bash
# 先拉取远程更新
git pull origin main

# 解决冲突后再推送
git push
```

### 部署失败

1. 访问 Actions 页面查看错误日志
2. 检查 `.github/workflows/deploy.yml` 配置
3. 确认 GitHub Pages 设置正确

### 网站未更新

1. 等待5分钟后刷新页面
2. 清除浏览器缓存（Ctrl + F5）
3. 检查部署是否成功完成
